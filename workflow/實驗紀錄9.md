### 處理步驟

**F-H4 trade-off 評估（3 方案）**：

| 方案 | 達成「terraform 偵測得到 image 變更」？ | 維護性（§0.3 #1） | blast radius | 結論 |
|---|---|---|---|---|
| (c) `apply -replace=...` | ✅（強制 re-create） | ❌ 靠部署者每次記得帶 magic flag；忘了跑純 apply 又靜默回舊 image；destroy+create 比 in-place 重 | 只動 build script + README | 否決（把 fix 藏在 operator 記憶裡 = 未來債） |

| **★ 採用：唯一 tag via 既有 var** | ✅（image_uri 字串隨每次部署改變 → in-place update） | ✅ 純 `terraform apply` 即生效、無 magic flag；tag = 人看得懂的 git SHA | **不動任何 .tf**（只動 build script + README + .gitignore） | **採用** |
