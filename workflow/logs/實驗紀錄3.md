  - mcp Phase M-1（通用化下載清洗）若沿用 ai v2 邏輯會踩同樣的雷

- **建議 mcp 採取的策略**：
  1. **M-1 下載/清洗工具設計時就支援多 data type**（M03A、M06A 都能下）
  2. **MCP tool 設計時區分**：`get_traffic_flow()` 用 M03A、`get_od_flow()` 用 M06A
  3. **RAG 知識庫補一句**：「M06A 不適合車流量預測，請用 M03A」（給未來查詢的使用者）
