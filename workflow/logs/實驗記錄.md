## [2026-04-14 02:30~02:50] 事件紀錄：Bucket 誤判與環境變數小寫問題

### 事件 A — boto3 找不到 credentials
- **現象**：直接 `python -c "import boto3; s3.list_objects_v2(...)"` 報 `NoCredentialsError`。
- **原因**：Python 子行程沒繼承 `.env`，需要先顯式載入。
- **處理**：在 `download_from_s3.py` 中自寫 `load_dotenv_file()`（仿 `upload_only_2025.py`）於 import boto3 前 setdefault 環境變數。

### 事件 B — `aws_session_token` 小寫無效
- **現象**：即使 `.env` 有 session token，boto3 仍然拒絕臨時憑證（Learner Lab 是 assumed-role，必須帶 token）。
- **原因**：`D:\p\112021134\.env` 第 4 行寫成小寫 `aws_session_token=...`，第 5 行大寫 `AWS_SESSION_TOKEN=`（空值）；boto3 只認大寫。`setdefault` 讀到小寫後沒補大寫版本，導致 boto3 拿到空 token。
- **處理**：`download_from_s3.py` 在 `load_dotenv_file` 之後補一段：
  ```python
  if os.environ.get("aws_session_token") and not os.environ.get("AWS_SESSION_TOKEN"):
      os.environ["AWS_SESSION_TOKEN"] = os.environ["aws_session_token"]
  ```
- **驗證**：`sts.get_caller_identity()` 成功回傳 `assumed-role/voclabs/user3852248=112021134@live.asia.edu.tw`。

### 事件 C — Bucket 名稱誤判 124 vs 134
- **現象**：`list_objects_v2(Bucket=<.env值>, Prefix='202603/')` 回傳 `KeyCount=0`，以為 S3 是空的。
- **除錯**：改用 `list_buckets()` 看到帳號下其實有 **兩個** bucket：
  - `112021124trafficdatacollectionsyste`（空）
  - `112021134trafficdatacollectionsyste`（有 202603/ 週分子夾）
- **根因**：copy 到 `step0_s3_download/.env` 的那份 S3_BUCKET 寫成 `112021124`（與使用者學號 `112021134` 只差一碼，疑似手動 typo 或舊版 .env）。
- **處理**：把 `final_presentation/step0_s3_download/.env` 第 9 行改為 `S3_BUCKET=112021134trafficdatacollectionsyste`。
- **驗證**：`list_objects_v2(Prefix='202603/', Delimiter='/')` 列出 5 個週分子夾。

### 事件 D — 背景任務終端輸出看不到
- **現象**：`python download_from_s3.py 2>&1 | grep -v ... | tail -80` 在執行期間輸出檔空白。
- **原因**：`tail` 要等 stdin EOF 才 flush，`grep` 管線也有緩衝，下載尚未結束就看不到任何 print。
- **替代方案**：改以磁碟計數（`ls ... | wc -l` + `du -sh`）作為進度指標；未來若要即時 log，改寫成 `python script.py > log.txt 2>&1` 然後用 `tail -F log.txt` 或 `Monitor` 工具。
