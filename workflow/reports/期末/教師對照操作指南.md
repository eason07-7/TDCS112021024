# 教師對照操作指南 — 本機 + AWS 雙邊驗證

> 本指南供老師 / 助教**對照操作**用：一邊讀講稿、一邊照步驟在本機與 AWS Console 操作、確認本專題提交的內容皆為真實落地。

### A4. 翻一頁源碼 — `infra/lambda/src/index.ts`

**操作**：打開 `infra/lambda/src/index.ts`、找到 PARQUET_SCHEMA 區塊：

```typescript
const PARQUET_SCHEMA = {
  year: pl.Int32,
  month: pl.Int32,
  day: pl.Int32,
  weekday: pl.Int32,
  hour_0: pl.Int32,
  gantry_id_o: pl.Utf8,
  vehicle_type: pl.Int32,
  counts: pl.Int32,
  week_index: pl.Int32,
};
```

**講稿**：
> 「這是書面報告 §3.4 那個 bug 的修補。一開始我們沒明示資料格式、polars 把所有數字都猜成 Float64、寫進 Parquet 變成 DOUBLE、Athena 一查就跟 Glue 設定的 INT 對不起來、噴錯。修補方式就是在這裡明示 schema、告訴它『這 8 欄是整數』、問題就解了。**這個修補對應 git commit `a381a2d`**、等下到 AWS Console 看 Lambda image tag 開頭就是這串。」

**對照書面報告**：§3.4 Schema 型別漂移 — 抓 bug → root cause → 修好

### C6. Athena 當場跑一次

**看什麼**：結果應該顯示：

**講稿**：

### C8. SQS 看 broker 設定
